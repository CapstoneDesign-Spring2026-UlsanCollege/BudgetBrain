# Render Deployment Guide

BudgetBrain deploys as one Docker web service with one Render PostgreSQL database.

## Deploy

1. Push the repository to GitHub.
2. Open Render and create a new Blueprint from the repository.
3. Confirm the resources defined in `render.yaml`.
4. Apply the Blueprint. Render provisions PostgreSQL and generates `JWT_SECRET`.
5. Set `CLIENT_ORIGIN` to the final HTTPS web-service URL if it differs from `https://budgetbrain.onrender.com`.
6. Optionally set `PADDLEOCR_API_URL` and `PADDLEOCR_API_KEY`.
7. Open `/api/health`, then test registration, CRUD, search, and receipt upload.

## Build design

`Dockerfile` uses three stages:

1. Node 22 builds the React/Vite application.
2. Maven with Java 21 copies the React build into Spring Boot static resources and packages the JAR.
3. A Java 21 JRE runs the application as an unprivileged user.

Spring Boot serves both the SPA and `/api` routes, avoiding cross-origin production requests. The database host, port, database name, username, and password are injected from the managed Render database.

## Required environment variables

| Variable | Source |
|---|---|
| `DB_HOST`, `DB_PORT`, `DB_NAME` | Render PostgreSQL |
| `DB_USERNAME`, `DB_PASSWORD` | Render PostgreSQL |
| `JWT_SECRET` | Generated by Render |
| `CLIENT_ORIGIN` | Public web-service URL |

Do not commit secrets or local `.env` files.
