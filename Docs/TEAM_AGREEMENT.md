# Team Agreement - BudgetBrain

## Members
BudgetBrain Development Team, Ulsan College Spring 2026

## Communication
- Daily standup at 10:00 AM
- GitHub Issues for task tracking
- Discord for instant messaging
- Weekly check-ins on Friday

## Code Standards
- Use consistent naming conventions (camelCase for JS/React)
- No external CSS frameworks (vanilla CSS only)
- Components should be under 200 lines
- Comment complex logic only
- Use existing color variables rather than hardcoding colors
- Test features before marking done

## Git Workflow
- Branch naming: `feature/feature-name`, `fix/bug-name`
- Commit messages: clear and descriptive
- No direct pushes to main without review
- Rebase before merging to keep history clean
- PR template must be used for all pull requests

## Responsibilities
- Each member owns their assigned features
- Update task status in GitHub Issues
- Report blockers immediately
- Test features before marking done
- Keep documentation up to date

## Deadlines
- Weekly sprint packets due Friday
- Code contributions by Thursday EOD
- Documentation updated with each sprint

## Conflict Resolution
- Discuss issues during standup
- Escalate to professor if unresolved
- Majority vote for technical decisions
